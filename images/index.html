Banco de Dados - SQL

O SQL é uma linguagem para gerenciar um SGBD (Sistema Gerenciador de Bancos de Dados). O Banco de Dados é composto de tabelas que são compostas por registros (linhas, tuplas) e campos (colunas, atributos).

O que são tabelas?

Podem ser definidas como uma coleção de entradas de dados relacionados e consiste em linhas e colunas. No sistema relacional, as tabelas se relacionam entre si utilizando determinadas linhas específicas. Uma tabela tem campos que representam atributos. Os atributos são os dados como nome, endereço, salário, data de nascimento, e assim por diante. Um campo é uma coluna específica em uma tabela que mantém informações específicas sobre cada registro.

Linha: É a entrada individual da tabela. Trata-se de um conjunto de campos relacionados que caracterizam os dados de uma entidade única.

Structured Query Language:

É a linguagem de consulta estruturada padrão para acesso ao BD. É usada em diversos sistemas de gerenciamento como Oracle, PostGreSQL, MySQL, SQL Server, Sybase, DB2, etc. Cada sistema pode usar um dialeto do SQL, como T-SQL (SQL Server), PL/SQL (Oracle), JET SQL (Acess), etc.

Funções do SQL:

  • Permite acesso a dados em SGBD;
  • Permite definir os dados no BD e manipulá-los;
  • Pode ser embutida em outras linguagens usando módulos SQL, e bibliotecas;
  • Permite criar BD e excluir tabelas;
  • Permite a criação de viwes, stored procedures, e functions em um BD;
  • Permite configurar permissões de acesso em tabelas, procedimentos e visões.

Grupos de Comandos:

Data Definition Language: Lida com esquemas e descrições no BD, de como os dados devem residir no BD. É referente a ESTRUTURA do BD.

  • CREATE → para criar o BD e seus objetos (ex: tabela, índice, visualizações, procedimentos de armazenamento), além de funções e gatilhos.
  • ALTER → modifica um objeto já existente no BD.
  • DROP → exclui uma tabela inteira, uma visualização de tabela ou outro objeto do BD.
  • TRUNCATE → remove todos os registros de uma tabela, incluindo todos os espaços para registros serão excluídos.
  • COMMENT → adiciona um comentário ao dicionário de dados.
  • RENAME → renomear um objeto.

DML: Data Manipulation Language: Lida com a manipulação de dados e inclui as instruções SQL: mais comuns. É usada para armazenar, modificar, recuperar e excluir, atualizar dados no BD.

  • SELECT → recupera dados no BD.
  • INSERT → insere dados numa tabela.
  • UPDATE → atualiza dados existentes em uma tabela.
  • DELETE → exclui todos os registros de uma tabela no banco de dados.
  • Operação MERGE - UPSERT → (inserir ou atualizar).
  • CALL → chama um subprograma PL/SQL ou Java.
  • EXPLAIN PLAN → interkpretação do caminho de acesso aos dados.
  • TABELA de BLOQUEIO → controle de simultaneidade.

Para diferenciar esses dois grupos de comandos, trago a frase de um autor anônimo.
